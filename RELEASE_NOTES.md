# Release Notes

## v0.1.0

First public release of **Speedrunning Social Development: How We Can Improve How We Improve**.

This release presents a paradigm-shifting conceptual framework for improving how digital systems improve. Instead of optimizing only user-facing features through blunt proxies, the paper argues that platforms should optimize the mechanisms by which they discover failure, absorb criticism, reward contributors, redesign their metrics, and learn from both compliant and adversarial use.

### Included in this release

- canonical paper source in `paper.tex`
- canonical comparison figure in `figures/traditional_vs_speedrunning_development_approaches.png`
- frozen release PDF for reading and citation
- citation metadata in `CITATION.cff`
- Zenodo metadata scaffold in `.zenodo.json`

### Key themes

- user-satisfaction as the intended value
- contributor development and compensation
- leaderboard feedback flywheels
- glitch-hunting as product learning
- Human/Tool x Compliant/Adversarial hypertopology

### Release intent

This version is meant to be publicly citable, critique-seeking, and stable enough to anchor a Zenodo DOI. Future versions may refine definitions, tighten empirical positioning, and add stronger pilot pathways without changing the core thesis.
